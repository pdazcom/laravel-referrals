# Laravel Referrals v2.5.0 Release Notes

## Highlights

- Confirmed Laravel 13 and PHP 8.3/8.4 compatibility.
- Removed the `version` field from `composer.json` (was a pre-existing release hygiene issue; Packagist resolves versions from git tags, not from this field).

## Laravel 13 Compatibility Audit

All framework APIs used by this package were verified against Laravel 13:

| API | Status |
| --- | --- |
| `EventServiceProvider` (`Illuminate\Foundation\Support\Providers\EventServiceProvider`) | No changes — fully compatible |
| `AboutCommand::add()` | No changes — fully compatible |
| `Illuminate\Database\Eloquent\Casts\Attribute::get()` | No changes — fully compatible |
| `Symfony\Component\HttpFoundation\Cookie` | No changes — fully compatible |
| `Request::merge()`, `Request::cookie()` | No changes — fully compatible |
| `cookie()` global helper | No changes — fully compatible |
| Service provider publishing (`publishes`, `loadMigrationsFrom`) | No changes — fully compatible |
| Event listeners registered via `$this->app['events']->listen()` | No changes — fully compatible |

No source changes are required for Laravel 13 support. The existing `composer.json` constraint (`^13`) is sufficient.

## PHP version notes

- PHP 8.2 supports Laravel 9, 10, 11, and 12.
- PHP 8.3 and 8.4 support Laravel 10, 11, 12, and 13.
- Laravel 13 requires PHP ^8.3 (enforced by `orchestra/testbench ^11`). PHP 8.2 + Laravel 13 is not a supported combination and is excluded from the CI matrix.
